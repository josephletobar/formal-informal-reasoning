# Preregistered ABC Replication Protocol

**Version:** 1.0  
**Purpose:** confirmatory replication of the exploratory Gemma 2B attribution-graph results  
**Status:** ready for timestamping before the next GPU run

## 1. Hypotheses

### Primary hypothesis

For a behaviorally calibrated computation, a source prompt using the same underlying computation will produce a larger target gold-minus-foil improvement, or a larger held-out causal rescue, than a source prompt matched for surface form but requiring a different computation.

### Mechanistic hypothesis

Sparse features and multilayer paths selected on discovery examples will recur on held-out examples sharing the underlying computation across ABC forms. Their intervention effects will exceed active same-position feature controls and wrong-operation controls.

### Falsification criteria

The mechanistic hypothesis is considered unsupported if any of the following occur in the preregistered primary analysis:

1. recurrent candidate features do not exceed active controls in held-out zero-ablation damage;
2. path transfer is not larger for same-computation targets than wrong-operation targets;
3. source activation transfer fails to improve a corrupted same-computation target more than the matched control; or
4. graph recurrence is explained equally well by surface form, answer identity or generic answer-delimiter features.

No exploratory subgroup may override a failed primary contrast.

## 2. Benchmark

The deterministic manifest is generated by `make_preregistered_benchmark.py` and saved under `benchmark_v1/`. It contains 360 rows:

- families: addition, subtraction and multiplication;
- forms: explicit, applied and implicit;
- eight fixed operand pairs;
- 36 direct discovery rows and 36 direct confirmation rows per family;
- four controls per direct row: surface/wrong computation, near miss, same-answer control and unrelated.

All direct and control gold answers are integers from 0 through 9. Any item whose exact model tokenization does not produce a one-token gold answer is excluded from the strict next-token causal objective and retained only in the behavioral generation report.

Discovery uses examples 0-3 and confirmation uses examples 4-7. The split is fixed before any model or graph inspection.

## 3. Behavioral calibration

Before mechanistic selection, evaluate all direct and control rows with greedy generation and the exact gold-minus-foil logit objective. Report both generation accuracy and objective margins. A family is considered calibrated if:

- direct explicit accuracy is at least 70%;
- at least 80% of direct rows have a positive gold-minus-foil margin; and
- no form has fewer than 50% positive margins.

If a family fails calibration, it remains in the behavioral report but is excluded from confirmatory mechanistic claims. This rule is applied before looking at graph candidates.

## 4. Attribution graph analysis

Use `google/gemma-2-2b-it` with the validated Gemma Scope replacement models at the same 26 layers as the exploratory run. Fix the following settings before running:

- answer-logit target: exact gold token;
- top answer parents per graph: 80;
- maximum feature nodes: 384;
- batch size: 1;
- CPU offload for graph construction;
- no graph-specific manual feature selection.

For each family, select candidates using discovery graphs only. A candidate must appear in at least three distinct discovery graphs and at least two ABC forms. Freeze the top 12 by graph count, then summed absolute edge attribution. Record all ties by a deterministic feature-index sort.

Primary representation endpoint: pairwise Jaccard overlap of top-parent sets. The preregistered contrast is same-computation/different-form versus same-surface/same-form controls. Report exact paired bootstrap intervals clustered by latent operand pair.

## 5. Held-out causal analysis

For every frozen candidate found at an exact graph-reported location in confirmation graphs:

1. record the clean gold-minus-foil margin;
2. zero the candidate feature;
3. replace it with its negative original activation;
4. apply the same interventions to one active noncandidate feature at the same layer and position.

The primary causal endpoint is recurrent-candidate zero-ablation damage minus active-control zero-ablation damage. Positive values support candidate necessity. The primary test is a paired bootstrap over graph locations, with a two-sided 95% interval and a predeclared permutation p-value.

## 6. Path and transfer analysis

From discovery graphs, freeze the rank-1 path per selected addition example. For confirmation targets, compare:

- zeroing the entire path;
- injecting source activations from a same-computation source;
- injecting source activations from a surface-matched wrong-operation source;
- injecting a matched random active path control.

The primary sufficiency endpoint is change from the corrupted target margin toward the clean target margin. A rescue is counted only when the clean target is initially correct or has a positive gold-minus-foil margin and the transfer increases that margin. Rescue after an intervention that improves the target is not counted as evidence of sufficiency.

## 7. Component analysis

Only after feature/path candidates are frozen, test at most:

- four attention heads selected from discovery data;
- two MLP blocks selected from discovery data;
- two individual MLP channels per selected block if channel-level evidence is available.

Component effects are tested on confirmation prompts at the exact final query position and at the algorithm-relevant source position. Attention analysis reports QK score, attention weight, OV answer-direction projection and source-position ablation separately.

## 8. Statistical analysis

All primary intervals use a cluster bootstrap over latent operand pairs, not independent row resampling. Secondary intervals use a normal approximation only for continuity with earlier reports. Every raw row remains available. Multiple secondary contrasts are controlled with Benjamini-Hochberg false-discovery-rate correction. Results are reported separately by family and form before any pooled summary.

The primary claims are:

1. same-computation versus surface/wrong-computation behavioral transfer;
2. same-computation/different-form graph overlap versus matched controls;
3. held-out recurrent-candidate versus active-control necessity;
4. same-computation path rescue versus wrong-operation path rescue.

No claim of a discrete module is permitted unless all four primary contrasts are positive, the candidate survives held-out active controls, and the effect replicates in an independent model or independent transcoder bundle.

## 9. Reproducibility

Before running the replication, record:

- git commit or file hashes for all scripts;
- model snapshot and transcoder hashes;
- package versions from `environment-lock.txt`;
- benchmark manifest hash;
- hardware and peak memory;
- start and end timestamps.

The protocol is intentionally stricter than the exploratory run. Its purpose is to make a null result informative and a positive result credible enough to support a serious journal submission.
